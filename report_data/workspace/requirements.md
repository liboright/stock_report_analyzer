# deep_research_report_skill 需求文档

## 一、项目背景

- **目的**：基于A股上市公司年报自动生成深度分析报告
- **数据来源**：仅使用年报数据（不使用网络搜索）
- **报告用途**：投资分析
- **当前数据**：宁德时代2023-2025年度报告
- **输出路径**：md/{公司名}/output/

## 二、功能需求

### 2.1 报告生成四阶段

#### 第一阶段：了解公司业务及行业情况
- 输入：公司名
- 处理：
  1. 读取该公司最近年度（如2025年）「第三节 管理层讨论与分析」的前两个小节
  2. 基于最近年度年报，先形成公司业务和行业的基本情况
  3. 由近到远逐年阅读之前年度（如2024、2023年）的对应章节
  4. 逐年补充行业变化、公司业务演进的数据
- 输出：`{公司名}_业务概况.md`

#### 第二阶段：数据搜集与拼接
- 输入：公司所有年份年报
- 处理：
  1. 按模版要求的分析维度，提取各年年报中的相关数据
  2. 将分散的数据按年份拼接，形成时间序列
  3. 数据类型包括：财务指标、产能产量、人员规模、市场份额等
- 输出：`{公司名}_数据汇总.md`

#### 第三阶段：逐章节写报告
- 输入：业务概况 + 数据汇总 + 报告模版文件
- 处理：
  1. 读取模版文件路径
  2. 按模版章节顺序，逐章生成内容
  3. 每章需要：引用拼接后的数据、结合业务理解进行分析、形成投资观点
- 输出：各章节草稿，保存在 `output/research_file/chapters/`

#### 第四阶段：整合与审阅
- 输入：各章节草稿
- 处理：
  1. 整合所有章节
  2. 审阅逻辑连贯性、数据一致性、表述清晰度
  3. 检查对读者不利的表述
- 输出：`{公司名}_深度报告_{日期}.md`

### 2.2 文件结构

```
md/{公司名}/
├── input/                              # 原始年报MD文件
│   └── {公司名}{年份}年度报告/
│       ├── 1_第一节 重要提示、目录和释义.md
│       ├── 2_第二节 公司简介和主要财务指标.md
│       └── ...
└── output/                              # 生成的研究报告
    ├── navi/                           # 年报章节导航JSON
    ├── research_file/                   # 中间研究文件
    │   ├── {公司名}_业务概况.md
    │   ├── {公司名}_数据汇总.md
    │   └── chapters/                    # 各章节草稿
    └── final/                          # 最终深度报告输出
```

### 2.3 Skill 核心模块结构

```
deep_research_report_skill/
├── __init__.py                    # Skill入口
├── phase1_business.py             # 第一阶段：业务了解
├── phase2_data.py                 # 第二阶段：数据拼接
├── phase3_writing.py              # 第三阶段：逐章写作
├── phase4_review.py               # 第四阶段：整合审阅
├── template/                      # 报告模版
│   └── investment_report.md       # 投资分析报告模版
└── utils/
    ├── file_loader.py             # 年报文件读取
    └── data_merger.py             # 数据合并工具
```

## 三、技术约束

1. **不使用网络搜索**：仅基于本地年报数据
2. **模版文件位置**：`{skill目录}/template/investment_report.md`
3. **中间文件**：保存在 `md/{公司名}/output/research_file/` 下
4. **最终报告**：保存为 `md/{公司名}/output/final/{公司名}_深度报告_{日期}.md`
5. **数据拼接**：以表格形式存储，便于后续引用

## 四、验证方式

1. 用宁德时代2023-2025年报数据，生成完整的深度报告
2. 检查各阶段输出文件是否正确生成
3. 检查最终报告的逻辑连贯性和可读性
