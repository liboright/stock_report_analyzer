## 错误分类

### 1. 依赖缺失错误（最常见）

**触发**：stage0 或 stage1 产物缺失。

**症状**：
```
FileNotFoundError: [Errno 2] No such file or directory: '宁德时代/md/research_file/参考资料/宁德时代_三年综合数据.md'
```

**处理**：硬失败，提示用户按正确顺序运行。

**错误消息模板**：
```
前置产物缺失（{N} 个）：
  - 宁德时代/md/research_file/参考资料/宁德时代_三年综合数据.md
  - 宁德时代/md/research_file/宁德时代_业务概况.md

stage2 依赖 stage0 + stage1。请按以下顺序运行：
  1. @stage0_data_aggregation 宁德时代     # 生成三年综合数据
  2. @stage1_business_understanding 宁德时代 # 生成业务概况 + 行业分析
  3. @stage2_generate_report 宁德时代        # 生成本报告
```

### 2. 文件读取错误

**触发**：产物存在但无法 Read（如权限、文件锁、损坏）。

**症状**：
```
Error: Failed to read file ...
```

**处理**：
- 单个文件读取失败 → 立即硬失败（这是依赖检查的一部分）
- 不要尝试"绕过"或"部分运行"

### 3. 最近一期年报定位失败

**触发**：stage1 业务概况已生成，但 stage2 找不到最近年份的年报目录。

**症状**：
```
FileNotFoundError: 宁德时代/md/clean/宁德时代2025年年度报告/管理层讨论/
```

**处理**：
- 这是**非致命错误**——本 skill 仍可生成报告，但**第七节（最近一期财务深度解读）的内容深度会降低**
- 降级策略：第七节只从 stage0 参考资料 + stage1 业务概况里抽取最近一年的数据，不再从年报原文捞事件
- 在 Session Summary 里标注 "第七节降级：未找到最近一期年报目录"

### 4. SubAgent 调用失败

**触发**：`report_writer` 或 `general-purpose` (review) 返回错误或超时。

**症状**：
```
Error: SubAgent call failed
```

**处理**：
- 立即硬失败
- 错误消息中明确告诉用户：报告未生成；stage0 + stage1 产物完好可重试

### 5. 章节缺失错误

**触发**：某版 SubAgent 生成的报告缺失 9 节中的任一节。

**症状**（来自第九阶段验证）：
```
RuntimeError: 报告缺失必备章节：['新业务', '最近一期财务深度解读']
```

**处理**：
- **不重跑 SubAgent**（太贵）
- **用 Edit 工具修复**：在报告末尾追加缺失章节，prompt 模板骨架可复用
- 如修复 3 次仍不达标 → 硬失败，提示用户手动补全

### 6. 数据缺失错误

**触发**：模板中某些字段在 stage0 参考资料 + stage1 产物中均未披露。

**症状**：报告中出现"未披露"。

**处理**：
- 这是**正常现象**，不视为错误
- 模板要求：缺失字段必须写"未披露"，**不能留空也不能编造**
- 验证阶段不检查"未披露"字段，只检查 9 节是否存在

### 7. 主线生成失败（v2 新增）

**触发**：`report_writer` (mainline mode) 未能产出有效主线文档。

**症状**：
- 返回内容非 markdown
- 主线文档结构不完整（缺 9 节论点 / 横向主题 / 关键数据点）

**处理**：
- **不重跑**——主线质量会因重试而下降（多样性损失）
- **用 v1 + 3 份原始材料** 退化为**单次端到端模式**（旧 v1 行为）：跳过主线，直接用 v1 作为最终版
- 在 Session Summary 标注 "主线生成失败，已降级为单次端到端模式（使用 v1 直接落盘）"
- 同时在最终报告顶部注明 "本报告为单次生成版本，未经过主线指引重写与审查循环"

### 8. 审查 agent 失败（v2 新增）

**触发**：`general-purpose` (review) 返回错误、超时，或返回的 JSON 无法解析。

**症状**：
```
JSONDecodeError: Expecting value
Error: SubAgent call failed
```

**处理**：
- **不重试审查**（审查是一次性评估，重试意义不大）
- **降级**：跳过润色循环，使用 v2 作为最终版
- 在 Session Summary 标注 "审查 agent 失败，已跳过润色循环（使用 v2 直接落盘）"

### 9. 审查未通过 + 达到 max_polish_rounds（v2 新增）

**触发**：连续 `max_polish_rounds` 轮润色后审查仍未通过。

**症状**：
- v3 / v4 审查仍报 critical_issues 非空 或 score < 7
- 循环退出条件: current_round = 2 + max_polish_rounds

**处理**：
- **不硬失败**——接受"尽力而为"原则
- 使用最后一版（v_{final_round}）作为最终版落盘
- 在 Session Summary 标注 "审查未完全通过，残留 N 个 critical_issue，最终使用 v{F} 落盘"
- 同时在最终报告顶部加可读性提示 "本报告经过 F-1 轮润色，仍有 N 项待优化建议，详见 _drafts/review_v{F-1}.json"
- **不静默失败**：用户必须能看到"还有问题"的信号

### 10. 润色产物异常（v2 新增）

**触发**：润色轮次产生的报告版本异常（如 9 节缺失、数据与 v_n 矛盾、引入投资建议）。

**症状**：
- 验证阶段 9 节缺失（见错误 5）
- 内容检查发现数据与 v_n 矛盾（如营收数字变了）
- 内容检查发现"目标价 / 评级"等投资建议字样

**处理**：
- **9 节缺失**：用 Edit 工具修复（见错误 5）
- **数据矛盾**：用 Edit 工具从 v_n 恢复正确数字
- **投资建议字样**：用 Edit 工具删除（不允许投资建议是硬规则）
- 修复后直接作为最终版，**不再进入下一轮润色**

## 错误传播规则

| 错误类型 | 失败粒度 | 是否硬失败 | 重试策略 |
|----------|----------|------------|----------|
| 依赖缺失 | 整体 | ✅ | 用户修复依赖后重试 |
| 文件读取失败 | 整体 | ✅ | 用户修复后重试 |
| 最近一期年报缺失 | 第七节降级 | ❌ | 降级运行 |
| SubAgent 调用失败（v1） | 整体 | ✅ | 用户重试 |
| 章节缺失 | 单版本 | ❌ | Edit 修复 |
| 数据缺失 | 字段 | ❌ | 用"未披露" |
| 主线生成失败 | 整体 | ❌ | 降级为单次端到端 |
| 审查 agent 失败 | 润色循环 | ❌ | 跳过循环，v2 落盘 |
| 审查未通过 + 达到 max | 整体 | ❌ | v_{F} 落盘 + 标注 |
| 润色产物异常 | 单版本 | ❌ | Edit 修复 |

## 重试建议

| 错误 | 是否值得重试 | 重试前必做 |
|------|--------------|------------|
| 依赖缺失 | ❌ | 用户先跑 stage0 + stage1 |
| 文件读取失败 | ❌ | 用户检查文件权限 |
| 最近一期年报缺失 | ❌（已降级） | — |
| SubAgent 调用失败 | ✅（1 次） | 检查 prompt 是否超长 |
| 章节缺失 | ✅（用 Edit 修复） | 不重跑 SubAgent |
| 主线生成失败 | ❌ | 降级为单次端到端 |
| 审查 agent 失败 | ❌ | 跳过润色循环 |
| 审查未通过 + 达 max | ❌ | 接受 v_{F} 落盘 |
| 润色产物异常 | ✅（用 Edit 修复） | 不重跑 SubAgent |

## 调试建议

如反复失败，按以下顺序排查：

1. **检查依赖**：3 个产物文件是否都存在且非空？
2. **检查文件大小**：stage0 参考资料应 ≥ 10 KB，stage1 业务概况应 ≥ 8 KB
3. **检查最近一期年报**：定位的年份是否正确？目录是否存在？
4. **检查 prompt 长度**：4 份材料总字符数应 < 100 KB
5. **检查审查报告**：`_drafts/review_v{N}.json` 是否合法？pass/fail 逻辑是否正确？
6. **检查 v_n 与 v_{n+1} 数据一致性**：v_{n+1} 是否偷偷改了 v_n 的数字？
7. **查看 LLM 日志**：`{公司}/output/log/llm_log_*.txt`

## 降级策略总结

| 阶段 | 正常路径 | 降级路径 |
|------|----------|----------|
| 最近一期年报 | 读 2-3 个关键章节 | 第七节用 stage0+stage1 凑 |
| 主线生成 | mainline mode | 用 v1 + 原始材料，直接跳到 v1 落盘（单次端到端） |
| 审查 agent | review + 条件润色循环 | 跳过审查 + 润色循环，用 v2 落盘 |
| 审查未通过 + 达 max | 强制落盘最后一版 | 落盘 + 标注残留问题 |
| 润色产物异常 | 修复后继续 | 修复后直接落盘，跳过剩余轮次 |
