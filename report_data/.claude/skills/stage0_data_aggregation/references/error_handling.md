# 错误处理

stage0_data_aggregation skill 运行时各类异常的处理规则。

## 章节读取

| 条件 | 动作 |
|------|------|
| Read 成功 + `len(content) >= 200` | 调 data_extractor 抽取 |
| Read 成功 + `len(content) < 200` | 跳过，记 `[skip-short]`，继续 |
| Read 失败（文件不存在/编码错误）| 跳过该章节，记 warning，继续下一章 |
| 章节含「详见本报告第 X 页」引用 | 同步 Read `by_section/行业分析/` 对应文件，调 SubAgent 抽取 |

## SubAgent 失败

| 失败类型 | 处理 |
|---------|------|
| Timeout / network error | 重试 1 次，仍失败 → 跳过该章节，记 warning，继续下一章 |
| Markdown 代码块缺失 | 跳过该章节，记 warning |
| Markdown 代码块但内容为空 | 跳过，记 `[skip-empty]` |
| SubAgent 返回「无可用数据」 | 接受，记 `[no-data]` |

**重要**：单章节失败不应阻断整次 skill 运行。其他章节继续处理。

## 模板找不到

- fallback 到**纯文本聚合模式**（无固定结构，按章节名直接堆叠抽取内容）
- 在文档头部标注「fallback 模式」

## 公司名错误

如果 `{公司}/md/clean/` 下找不到对应公司目录：
- 报错：`公司数据目录不存在：{公司}/md/clean/`
- 提示用户检查公司名拼写

## 年份缺失

如果 `{公司}/md/clean/` 下没有任何 `{公司}{年}年年报/` 子目录：
- 报错：`未找到任何年报数据目录`
- 提示用户确认公司数据已就绪

## 路径错误

| 症状 | 原因 | 修复 |
|------|------|------|
| `No such file or directory` | cwd 不对 | `cd REPORT_DATA_PATH` 或用绝对路径 |
| 写文件无反应或权限错误 | M3 sandbox 拦截 `D:/...` 路径 | 改用 `/d/...` 前缀 |
| bash 输出乱码 | Windows 编码问题 | 改用 `Glob` 工具代替 `ls` |

## 数据缺失值处理

年报未披露某指标时：

- 抽取 SubAgent 输出：单元格写「未披露」或文字「年报未披露该数据」
- **不要留空**（避免后续解析歧义）
- **不要外推**（禁止根据其他年份推断）
- **禁止网络搜索补充**

## 跨年章节名差异

如果同一概念在 2023/2024/2025 年报中章节名不同（如「主要业务」vs「主要经营情况」）：
- **不要强制合并**——保留各自独立章节
- 在文档头部标注：「本章在 {年份列表} 中章节名分别为 {name_2025} / {name_2024} / {name_2023}」
- 让读者自行关联

## 产物文件已存在

- 默认行为：**覆盖**（用最新数据替换）
- 不保留历史版本（避免磁盘膨胀）
- 如需保留版本化，使用 git 管理

## 日志记录失败

| 失败类型 | 处理 |
|---------|------|
| log 文件写入失败 | 警告并继续，不阻断主流程 |
| log_agent_call 超 20 万字符 | 自动截断并标注「[truncated]」 |
