# 示例：宁德时代 2023-2025 三年数据聚合

实际跑通的完整案例（2026-06-22）。

## 输入

```
@stage0_data_aggregation 宁德时代
```

## 实际步骤

1. **扫描** → 41 个章节文件，3 个年份（2025/2024/2023）
2. **逐章节 Read** → 全部 41 个章节（其中 1 个 < 200 字符被 skip-short）
3. **调 data_extractor SubAgent** → 每个有效章节调 1 次 general-purpose SubAgent
4. **聚合** → 按章节名合并跨年数据，生成动态 TOC
5. **落盘** → 1 个 markdown 文档

## 产物

| 文件 | 路径 | 字节数 | 章节数 | 表数 |

## 备注

- 本次跑的是「逐章节抽取」模式：每个 (年份, 章节) 调 1 次 general-purpose SubAgent
- 实际 SubAgent 调用数：~80 次（40 章节 × 2 年补充材料合并后）
- 相比"端到端综合抽取"，逐章节模式更稳定（每个 SubAgent 任务更小）
